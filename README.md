# 🚀 My FastAPI App

A simple Employee Management REST API built with **FastAPI**, **SQLAlchemy**, and **SQLite**.

## 📁 Project Structure

```
my-fastapi-app/
├── main.py              # FastAPI app & route definitions
├── database.py          # SQLAlchemy engine, session & DB dependency
├── models.py            # SQLAlchemy ORM model (EmployeeDB)
├── schemas.py           # Pydantic schemas (EmployeeCreate, EmployeeResponse)
├── crud.py              # Database CRUD operations
├── pydantic_demo.py     # Pydantic demo with a User endpoint
├── .gitignore
└── README.md
```

## 🛠️ Tech Stack

- [FastAPI](https://fastapi.tiangolo.com/) — Modern, high-performance Python web framework
- [SQLAlchemy](https://www.sqlalchemy.org/) — Python SQL toolkit & ORM
- [SQLite](https://www.sqlite.org/) — Lightweight file-based database
- [Pydantic](https://docs.pydantic.dev/) — Data validation using Python type hints
- [Uvicorn](https://www.uvicorn.org/) — ASGI server to run the app

## 📦 Setup

```bash
# Clone the repo
git clone https://github.com/aaranarc/my-fastapi-app.git
cd my-fastapi-app

# Create a virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install fastapi uvicorn pydantic sqlalchemy
```

## ▶️ Run the App

```bash
uvicorn main:app --reload
```

The API will be available at `http://127.0.0.1:8000`

Interactive docs at `http://127.0.0.1:8000/docs`

> **Note:** The SQLite database (`employees.db`) is auto-created on first run.

## 📡 API Endpoints

### Employee API (`main.py`)

| Method   | Endpoint                      | Description              |
|----------|-------------------------------|--------------------------|
| `GET`    | `/employees`                  | Get all employees        |
| `GET`    | `/employees/{emp_id}`         | Get employee by ID       |
| `POST`   | `/employees`                  | Add a new employee       |
| `PUT`    | `/update_employee/{emp_id}`   | Update an employee       |
| `DELETE` | `/delete_employee/{emp_id}`   | Delete an employee       |

### Request Body (POST / PUT)

```json
{
  "name": "Aarana",
  "age": 25,
  "dep": "Engineering"
}
```

> `id` is auto-generated by the database — no need to pass it.

### Response

```json
{
  "id": 1,
  "name": "Aarana",
  "age": 25,
  "dep": "Engineering"
}
```

### Pydantic Demo (`pydantic_demo.py`)

| Method | Endpoint | Description          |
|--------|----------|----------------------|
| `GET`  | `/user`  | Returns a sample user |

## 🏗️ Architecture

```
Request → main.py (routes) → crud.py (DB logic) → models.py (ORM) → SQLite
                                                    ↑
                                              schemas.py (validation)
```

- **`database.py`** — Sets up the SQLAlchemy engine, session factory, and `get_db` dependency
- **`models.py`** — Defines the `EmployeeDB` SQLAlchemy table model
- **`schemas.py`** — Pydantic schemas for request validation (`EmployeeCreate`) and response serialization (`EmployeeResponse`)
- **`crud.py`** — All database operations (create, read, update, delete)
- **`main.py`** — FastAPI app with route handlers that use `Depends(get_db)` for DB sessions
